About the Site

Learning front-end development through technology analysis

This site was created as an academic project inspired by the Colombia 5.0 event. Its objective is to organize technical information about digital design, user interfaces, automation, and responsible innovation using HTML5, CSS3, JavaScript, and Bootstrap 5.

The main topic is the creation of video game user interfaces with Figma. The content highlights scalable design, reusable components, responsive structures, and the importance of professional ethics when technology affects people, data, and business decisions.

Conferences and Workshops

Technical talks selected for the project

The following summaries connect event topics with practical skills for systems engineering students.

Assigned Talk

Creating UI for Video Games with Figma

Speaker: Presenter from the analyzed event video.

Main topic: Efficient and scalable user interface design for video games.

The talk explains how Figma helps designers build flexible interfaces through the 9-slice technique, Auto Layout, frames, assets, variants, and reusable components. The presenter emphasizes that a solid design system improves consistency and saves time because changes made to a main component can be reflected in all its instances.

A central idea is that artificial intelligence will be more useful when designers provide clear, modular, and well-defined component systems. In that sense, the designer becomes a guide who prepares the structure that future tools can understand and reuse responsibly.

Keywords

Figma, UI, components, Auto Layout, variants, design systems, AI, 9-slice, frames, assets.

Selected Talk

Responsible AI and Business Technology

Speaker: Student-selected academic reflection.

Main topic: Ethical use of automation, data, and artificial intelligence in companies.

This topic focuses on how organizations can use technology without ignoring its social impact. Artificial intelligence can improve productivity, personalize services, and automate repetitive tasks, but it also requires transparency, data protection, and human supervision.

For future software developers, ethical responsibility means designing systems that are useful, secure, accessible, and fair. A technical solution should not only work correctly; it should also respect people, reduce risks, and support responsible decisions.

Keywords

Artificial intelligence, data privacy, automation, accessibility, security, social impact, professional responsibility.

Additional Information

Why design systems matter

Design systems organize visual decisions, interface components, and interaction states. In video game development, this helps create consistent screens, menus, buttons, health bars, and feedback elements across different devices.

Scalability

Reusable components allow a project to grow without losing visual order.

Consistency

Variants make it easier to manage states such as normal, active, and disabled.

Future AI Support

Well-structured systems can help intelligent tools generate clearer interface proposals.

Conclusion

How ethics is implemented in business technology

Ethics in business technology is implemented by designing systems that respect people, protect information, and make decisions transparent. Artificial intelligence should be used with human supervision, clear objectives, and mechanisms that reduce bias or unfair results.

Responsible data use means collecting only what is necessary, protecting private information, and explaining how data supports automated processes. Automation can improve efficiency, but it should not remove responsibility from developers, companies, or decision makers.

As a systems engineering student, I understand that professional responsibility is part of software quality. A good project should be functional, accessible, secure, and aware of its social impact. Technology has real effects, so developers must think beyond code and consider the people who will depend on the systems they build.
